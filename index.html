Here we have some basic HTML tags (commands).

The Absolute Basics -- How to structure the beginning of a web page.

Here are some basic HTML tags (commands), required for outlining a document. These are the absolute basics required to create a web page.

  1. Every HTML document must begin with the tag <!DOCTYPE html>. Then all the actual HTML (tags and text) must go between a <html> and a </html>. That is, the web page begins with a tag that describes the document type as HTML and a tag pair that contains all the actual HTML code and text.
  2. As we saw in the first item, most tags come in pairs: one tag "opens" the environment we are using, and the next tag "closes" that environment. There are exceptions to this pairing, but this pairing is the norm for HTML tags. (In the first item, we saw this pairing when <html> and </html> were used as a pair.)
  3. To make a comment in HTML, we use <! to start the comment, and --> to end a comment. In HTML, comments do not automatically end at the end of a line. It is generally good form to begin a comment with <!--, but the two hyphens are not required for a comment.
  4. The <header> and </header> tag pair define the web page's header. That is, they are meant to go around meta-information for the web page. The content you place within this tag pair will not actually appear on your web page, but may help your page render in the way you desire. The single most common use of the page header is for the <title> and </title> tag pair. This title tag defines your web page title, for display in your browser window or tab title.
  5. The <body> and </body> tag pair define the body of your web page. The text and HTML code you place in here will show up on your actual web page.

The sample code below shows an extremely minimal web page, using the above HTML tags.

<!DOCTYPE html>

<html>

<head>

<title>

Simple Web Page

</title>

<!-- This is a comment. The title above puts "Simple Web Page" in the browser title bar or browser tab title. -->

</head>

<body>

<p>This little bit of text goes on the actual web page.</p>

<!-- This is another comment...

...new line...still a comment...

...still a comment...

...comment ends here. -->

</body>

</html>

You may notice that there is some paired indenting in the example above. The indenting is just a common style preference -- indenting is not required to make the HTML display accurately. We can also notice that all of the tags in the example above come in pairs, except the initial tag ( <!DOCTYPE html>) at the beginning of the document.
